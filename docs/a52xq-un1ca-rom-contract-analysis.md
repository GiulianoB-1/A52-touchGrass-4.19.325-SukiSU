# A52xq UN1CA kernel-facing ROM contract

## Provenance

Input archive: `a52xq-rom-contract.zip`

SHA-256:

```text
27ff52128565b3f5d5f7d74f4076414a1d8085ecdab0faf569be872b14e8aca1
```

The archive contains 189 files:

- 135 vendor init `.rc` files
- 42 vendor VINTF XML files
- 1 ODM VINTF XML file
- 6 vendor kernel modules plus module metadata
- a 594-path firmware inventory

No private device identifiers are recorded in this report.

## VINTF and userspace level

The primary vendor manifest declares:

- device manifest target level 5
- vendor NDK level 30
- system SDK level 30
- 126 unique HAL package names across the collected vendor and ODM VINTF files

The HAL set spans Qualcomm and Samsung display, graphics, radio, Wi-Fi, audio, camera, biometrics, security, thermal, NFC, USB, power and media components. This does not mean all of them are required for the first 5.10 boot. It means later hardware enablement must preserve the kernel interfaces consumed by those vendor services.

## Legacy control-group contract

UN1CA's vendor init scripts explicitly create and use legacy control-group v1 interfaces:

- freezer under `/dev/freezer`
- cpusets under `/dev/cpuset`
- blkio under `/dev/blkio`
- schedtune under `/dev/stune`

The 5.10 device configuration must retain the corresponding legacy cgroup, freezer, cpuset, blkio and scheduler support expected by the Android 11-era vendor layer. A pure cgroup-v2-only configuration is not acceptable for this ROM contract.

## Vendor modules

The vendor module directory contains:

```text
llcc_perfmon.ko
mpq-adapter.ko
mpq-dmx-hw-plugin.ko
rdbg.ko
rmnet_perf.ko
rmnet_shs.ko
```

`modules.load` lists all six. `mpq-dmx-hw-plugin.ko` depends on `mpq-adapter.ko`; the other listed modules have no module-to-module dependency in `modules.dep`.

Every module has this stock Samsung kernel ABI identity:

```text
4.19.152-perf-28692722-abA526BXXSCGYC1 SMP preempt mod_unload modversions aarch64
```

All are signed with the same Samsung build-time autogenerated kernel key. They cannot be loaded into a 5.10 GKI kernel and must either be rebuilt from matching source against the selected 5.10 KMI or omitted/replaced.

The prior live Android dump showed an empty `/proc/modules`, so these six modules are not required to establish the first storage/init boot milestone. They can remain a later compatibility gate.

## GKI-related init remnants

Qualcomm vendor init scripts contain alternative module search paths under:

```text
/vendor/lib/modules/5.4-gki
```

These paths are used for some rmnet commands, but no matching `5.4-gki` module directory was present in the collected module payload. This is evidence of generic Qualcomm multi-kernel init code, not evidence that the A52 vendor image already provides a reusable GKI module set.

## Firmware contract

The collected inventory contains 594 firmware paths. Important families include:

- Lagoon IPA firmware: `lagoon_ipa_fws.*`
- Qualcomm A6xx GPU firmware and secure zap images
- WPSS/WLAN firmware selected by device and bootloader revision
- NFC firmware
- camera ICP firmware
- Bluetooth firmware
- the larger `vendor/firmware_mnt/image` modem, DSP and subsystem firmware set

Only paths were collected, not the firmware binary contents. For the first 5.10 port this is sufficient to identify expected request names. The running ROM can continue supplying the proprietary firmware from `/vendor` and `/vendor/firmware_mnt` as long as the ported drivers request the same names and preserve the expected subsystem interfaces.

## Kernel interfaces exposed to vendor init

The vendor init layer expects hundreds of Samsung and Qualcomm kernel nodes. Important groups include:

- UFS and block-device topology
- QSEECom, SMEM and secure-world communication
- IPA and rmnet networking
- USB gadget, Type-C role and Qualcomm SSUSB nodes
- subsystem restart and ramdump controls
- Wi-Fi boot and `/sys/wifi` Samsung nodes
- Samsung display, panel, backlight and mDNIe nodes
- STM/Samsung touchscreen and trusted-touch nodes
- fingerprint, NFC, vibrator, audio and camera nodes
- Samsung rawdump/ramdump partition access

These are later bring-up gates. They must not all be ported before the first diagnostic boot.

## Porting decision

The ROM contract supports the staged 5.10 plan:

1. Keep the official Android 12 GKI core pinned and reproducible.
2. Port only the boot-critical Lagoon chain first: DT, interrupt/timer, clocks, regulators, interconnect, SCM, SMMU, reserved memory, UFS, storage encryption, filesystems, console and ramoops.
3. Preserve the UN1CA header-v2 boot ramdisk and separate A52 DTBO layout.
4. Reach first-stage init or obtain a persistent kernel log before working on display or input.
5. Rebuild or replace all external modules against the final 5.10 source and KMI.
6. Add the vendor-facing Samsung and Qualcomm sysfs/device interfaces subsystem by subsystem.

## Current conclusion

The ROM contract dump is complete enough to begin source-level 5.10 port planning. No additional device dump is required before comparing the pinned GKI configuration and source against the A52 4.19 Lagoon hardware layer.
